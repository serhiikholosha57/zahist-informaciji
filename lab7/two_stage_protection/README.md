# Двоетапна система захисту з аналітикою

## Опис

Комплексна система захисту, що поєднує:
1. **AES-256 шифрування** - захист конфіденційності даних
2. **LSB-стеганографія** - приховування зашифрованих даних у зображенні

## Встановлення

```bash
pip install -r requirements.txt
```

## Використання

### 1. Захист файлу

```bash
python main.py protect <файл> <зображення> <вихідне_зображення> <пароль>
```

Приклад:
```bash
python main.py protect secret.txt cover.png stego.png mypassword123
```

### 2. Відновлення файлу

```bash
python main.py recover <стего_зображення> <вихідний_файл> <пароль>
```

Приклад:
```bash
python main.py recover stego.png recovered.txt mypassword123
```

### 3. Тестування з аналітикою

```bash
python main.py test <оригінальний_файл> <зображення> <пароль>
```

Приклад:
```bash
python main.py test secret.txt cover.png mypassword123
```

Цей режим:
- Виконує повний цикл захист→відновлення
- Перевіряє цілісність даних
- Генерує детальний звіт з метриками
- Зберігає звіт у файл `report.txt`

## Структура проекту

```
two_stage_protection/
├── main.py              # Головний модуль
├── encryption.py        # AES шифрування
├── steganography.py     # LSB стеганографія
├── analytics.py         # Модуль аналітики
├── requirements.txt     # Залежності
└── README.md           # Документація
```

## Метрики аналітики

Система збирає:
- Час виконання кожного етапу
- Розміри файлів до/після обробки
- Накладні витрати шифрування
- Використання ємності зображення
- Загальну статистику операцій

## Особливості

✓ Автоматичне послідовне застосування обох методів
✓ Повне відновлення оригінального файлу
✓ Перевірка цілісності даних
✓ Детальна аналітика ефективності
✓ Генерація звіту з рекомендаціями
✓ Демонстрація необхідності обох етапів для доступу

## Вимоги безпеки

- Використовуйте надійні паролі (мінімум 12 символів)
- Зберігайте пароль окремо від стего-зображення
- Для відновлення необхідні ОБА етапи:
  1. Екстракція зашифрованих даних з зображення
  2. Дешифрування з правильним паролем

Без будь-якого з цих етапів дані неможливо відновити.
