# Інструкція для демонстрації системи

## Підготовка

1. Встановіть залежності:
```bash
pip install -r requirements.txt
```

2. Створіть тестові файли:
```bash
python create_test_files.py
```

## Демонстрація 1: Повний цикл з аналітикою

```bash
python main.py test test_secret.txt test_cover.png TestPassword123
```

Ця команда:
- Шифрує файл `test_secret.txt`
- Приховує зашифровані дані в `test_cover.png`
- Екстрагує та дешифрує дані назад
- Перевіряє цілісність
- Генерує детальний звіт у `report.txt`

## Демонстрація 2: Покрокове використання

### Крок 1: Захист файлу
```bash
python main.py protect test_secret.txt test_cover.png output_stego.png MySecurePass123
```

### Крок 2: Відновлення файлу
```bash
python main.py recover output_stego.png recovered_secret.txt MySecurePass123
```

### Крок 3: Перевірка результату
```bash
cat recovered_secret.txt
```

## Демонстрація 3: Необхідність обох етапів

```bash
python demo_security.py demo_stego.png TestPassword123
```

Ця демонстрація показує:
1. Екстраговані дані без дешифрування нечитабельні
2. Неправильний пароль не дає доступу до даних
3. Обидва компоненти необхідні для успішного відновлення

## Результати демонстрації

### Метрики з звіту:
- Час шифрування: ~0.04 сек
- Час стеганографії: ~0.06 сек
- Використання ємності: ~0.2% (висока непомітність)
- Накладні витрати шифрування: ~12%
- Успішність перевірки цілісності: 100%

### Переваги системи:
✓ Подвійний захист: шифрування + стеганографія
✓ Швидка обробка
✓ Висока непомітність
✓ Гарантована цілісність даних
✓ Детальна аналітика операцій

## Структура згенерованих файлів

```
two_stage_protection/
├── test_secret.txt        # Оригінальний файл
├── test_cover.png         # Зображення-контейнер
├── demo_stego.png         # Стего-зображення з прихованими даними
├── report.txt             # Детальний звіт з аналітикою
└── recovered_secret.txt   # Відновлений файл
```

## Примітки

- Розмір зображення повинен бути достатнім для приховування даних
- Використовуйте надійні паролі (12+ символів)
- Стего-зображення візуально ідентичне оригіналу
- Система підтримує будь-які типи файлів
